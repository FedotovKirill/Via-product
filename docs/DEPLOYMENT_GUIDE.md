# 📘 Руководство по развёртыванию Via (на базе RHEL)

**Назначение:**  
Бот автоматически отслеживает изменения в задачах Redmine и отправляет уведомления в Matrix.  

Настоящее руководство содержит подробные инструкции по развёртыванию решения на системах с пакетным менеджером `dnf` (Red OS 8.0+, AlmaLinux 9, Rocky Linux 9, RHEL 9).

---

## 1. Подготовка виртуальной машины

Для развёртывания требуется виртуальная машина под управлением Linux с root-доступом.

**Требования к ВМ:**
- **Операционная система:** Red OS 8.0+, AlmaLinux 9, Rocky Linux 9 или RHEL 9
- **Доступ:** root или пользователь с правами `sudo`
- **Сетевые данные:**
  - **IP-адрес:** `<ваш_IP_адрес>`
  - **Пользователь:** `username` (или `root`)

---

## 2. Установка необходимого программного обеспечения

Все команды выполняются на виртуальной машине от имени пользователя `root` или с использованием `sudo`.

### 2.1. Обновление системы
```bash
dnf update -y
```

### 2.2. Установка базовых утилит
```bash
dnf install -y git nano
```

### 2.3. Установка Docker Engine и Docker Compose
```bash
dnf install -y docker-ce docker-ce-cli docker-compose
```
*Примечание:* Если пакеты Docker отсутствуют в стандартных репозиториях системы, выполните шаг 2.4.

### 2.4. Подключение официального репозитория Docker
```bash
dnf config-manager --add-repo https://download.docker.com/linux/centos/docker-ce.repo
```

### 2.5. Запуск и включение автозапуска Docker
```bash
# Включение автозапуска и запуск сервиса
systemctl enable --now docker

# Проверка статуса сервиса (должен отображаться как active (running))
systemctl status docker
```

### 2.6. Настройка прав доступа (без использования sudo)
```bash
# Добавление текущего пользователя в группу docker
usermod -aG docker $USER

# Применение изменений без перезапуска сессии
newgrp docker
```

### 2.7. Проверка установки
```bash
docker --version 
docker compose version
```

## 3. Настройка Firewall и SELinux

### 3.1. Открытие порта для веб-интерфейса
Если активен `firewalld`, выполните:
```bash
firewall-cmd --permanent --add-port=8080/tcp
firewall-cmd --reload
```

### 3.2. Настройка SELinux (при необходимости)
В случае проблем с монтированием томов Docker (редко) можно временно перевести SELinux в режим `permissive` для диагностики:
```bash
setenforce 0
sed -i 's/^SELINUX=enforcing/SELINUX=permissive/' /etc/selinux/config
```
**Рекомендация:** После успешного развёртывания верните режим в `enforcing` и настройте соответствующие политики SELinux.

## 4. Развёртывание проекта

### 4.1. Клонирование репозитория
```bash
# Создание рабочей директории (рекомендуется использовать /opt или домашний каталог)
mkdir -p ~/via
cd ~/via

# Клонирование репозитория
git clone git@github.com:forgebeard/Via.git
cd Via
```

### 4.2. Запуск (Zero-Config)
```bash
# Скрипт автоматически создаст .env и поднимет все сервисы
chmod +x deploy.sh
./deploy.sh
```

**Автоматические действия при первом запуске:**
1. Создание пустого `.env` (если отсутствует).
2. Загрузка необходимых образов (PostgreSQL, Python).
3. Сервис `init` автоматически заполнит `.env` случайными `POSTGRES_PASSWORD` и `APP_MASTER_KEY`.
4. Будут запущены база данных, веб-панель управления и бот.

> ⚠️ **Важно:** Сохраните `.env` — в нём находятся credentials для восстановления системы.

## 5. Первичная настройка через веб-интерфейс

Откройте в браузере ссылку:  
`http://<ваш_IP_адрес>:8080/setup`

### 5.1. Создание учётной записи администратора
Придумайте логин и пароль для входа в панель управления.

### 5.2. Авторизация
Используйте созданные учётные данные на странице `/login`.

### 5.3. Настройка интеграций
Перейдите в раздел **«Настройки»** (`/onboarding`):

1. **Раздел «База данных сервиса»**  
   Скопируйте значения `POSTGRES_PASSWORD` и `APP_MASTER_KEY` в безопасное место (для восстановления системы).

2. **Раздел «Параметры сервиса»**  
   - Укажите параметры **Redmine**: URL и API Key.  
   - Укажите параметры **Matrix**: Homeserver, Access Token, User ID.  
   - Нажмите **«Проверить доступ»** для проверки подключений.  
   - Нажмите **«Сохранить»**.

3. **Перезапуск бота** (для применения настроек):
   ```bash
   docker compose restart bot
   ```

## 6. Полезные команды управления

```bash
# Обновление системы
dnf update -y

# Просмотр логов бота в реальном времени
docker compose logs -f bot

# Перезапуск сервиса бота
docker compose restart bot

# Полная остановка и очистка (удаление БД, образов и томов)
docker compose down --rmi all -v

# Просмотр статуса контейнеров
docker compose ps
```

---

**Рекомендации по безопасности:**
- Используйте SSH-ключи вместо паролей.
- Регулярно обновляйте систему и Docker-образы.
- Храните чувствительные данные (`POSTGRES_PASSWORD`, `APP_MASTER_KEY`, токены) в защищённом хранилище.

В случае возникновения вопросов или ошибок обращайтесь к логам контейнеров (`docker compose logs`).
