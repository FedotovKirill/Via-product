# 📘 Инструкция по развёртыванию Via на чистой ВМ

Бот автоматически отслеживает изменения в задачах Redmine и отправляет уведомления в Matrix.

---

## 1. Подготовка ВМ

Тебе понадобится виртуальная машина с Linux (Ubuntu 22.04/24.04 или Debian 11/12) и root-доступом.

**Данные:**
- **IP:** `<твой_ip_адрес>`
- **Пользователь:** `redadmin` (или `root`, тогда замени в командах)

---

## 2. Установка Docker и Docker Compose

Выполни эти команды на ВМ (от root или через `sudo`):

```bash
# Обновляем пакеты
apt update && apt upgrade -y

# Устанавливаем Docker (официальный скрипт)
curl -fsSL https://get.docker.com -o get-docker.sh
sh get-docker.sh

# Добавляем пользователя в группу docker (чтобы не писать sudo)
usermod -aG docker $USER
# Если ты root, замени $USER на redadmin: usermod -aG docker redadmin

# Включаем автозапуск Docker
systemctl enable docker --now

# Проверяем версию compose (должна быть 2.x)
docker compose version
```

> ⚠️ **Важно:** После добавления пользователя в группу нужно **перелогиниться** (выйти и зайти по SSH снова), чтобы права применились.

---

## 3. Клонирование проекта

На ВМ:

```bash
# Создаём директорию
mkdir -p ~/Документы/pet_projects/redMat
cd ~/Документы/pet_projects/redMat

# Клонируем репозиторий
git clone git@github.com:forgebeard/Via.git
cd Via
```

---

## 4. Запуск (Zero-Config ✨)

Больше ничего настраивать не нужно. Просто запусти:

```bash
docker compose up --build -d
```

**Что произойдёт автоматически:**
1. Docker скачает образы (Postgres, Python).
2. Сервис `init` сгенерирует файл `.env` со случайными `POSTGRES_PASSWORD` и `APP_MASTER_KEY`.
3. Поднимется база данных, админка и бот.

---

## 5. Первичная настройка (через браузер)

Открой на своём компьютере:
👉 `http://<IP_ТВОЕЙ_ВМ>:8080/setup`

### Шаг 5.1: Создание администратора
Придумай логин и пароль для входа в панель управления.

### Шаг 5.2: Вход в систему
Используй созданные данные на странице `/login`.

### Шаг 5.3: Настройка подключений
Перейди в раздел **"Настройки"** (`/onboarding`):

1. **Раздел "База данных сервиса":**
   - Скопируй `POSTGRES_PASSWORD` и `APP_MASTER_KEY` в надёжное место (на случай восстановления).
   - Эти пароли сгенерированы автоматически при первом запуске.

2. **Раздел "Параметры сервиса":**
   - Введи данные **Redmine**: URL и API Key.
   - Введи данные **Matrix**: Homeserver, Access Token, User ID.
   - Нажми **"Проверить доступ"**, чтобы убедиться, что подключения работают.
   - Нажми **"Сохранить"**.

3. **Перезапусти бота** (чтобы он подхватил настройки):
   ```bash
   docker compose restart bot
   ```

## 6. Настройка быстрого деплоя (RSync)

Чтобы не пушить в Git каждый раз, когда ты меняешь одну строчку кода на ноутбуке, настрой быструю синхронизацию.

### На твоём ноутбуке:

**1. Генерируем SSH-ключ (если нет):**
```bash
ssh-keygen -t ed25519 -C "via-deploy"
# На все вопросы жми Enter
```

**2. Копируем ключ на ВМ:**
```bash
ssh-copy-id redadmin@<IP_ТВОЕЙ_ВМ>
# Введи пароль от ВМ один раз
```

**3. Команда для деплоя изменений:**
Теперь, когда ты поправил код локально, просто выполни в папке проекта:
```bash
rsync -avz \
  --exclude='.git' \
  --exclude='venv' \
  --exclude='__pycache__' \
  --exclude='.pytest_cache' \
  --exclude='data/' \
  --exclude='.env' \
  ./ \
  redadmin@<IP_ТВОЕЙ_ВМ>:/home/redadmin/Via/
```

**4. Перезапуск сервисов на ВМ:**
```bash
cd ~/Via && docker compose up -d --build admin
# или bot, в зависимости от того, что менял
```

## 7. Перегенерация паролей

Если нужно сменить пароль БД или мастер-ключ:

**Через GUI (рекомендуется):**
1. Настройки → **База данных сервиса** → кнопка **"Сгенерировать новые"**
2. Подтвердите действие
3. Перезапустите контейнеры: `docker compose restart postgres bot admin`

**Вручную (через скрипт):**
```bash
docker compose run --rm -e REGENERATE=1 init
docker compose restart postgres bot admin
```

## Чек-лист проверки работоспособности

- [ ] `docker compose ps` показывает все сервисы `Up` (кроме `init` — он `Exited (0)`).
- [ ] Админка открывается по `http://<IP>:8080`.
- [ ] В логах бота (`docker compose logs bot`) нет ошибок `Connection refused`.
- [ ] Тестовое сообщение из админки доходит до пользователя в Matrix.
- [ ] Поиск пользователей Redmine работает (возвращает список).

## Полезные команды

```bash
# Логи всех сервисов
docker compose logs -f

# Логи конкретного сервиса
docker compose logs -f bot
docker compose logs -f admin

# Перезапуск сервиса
docker compose restart bot

# Полная остановка (данные БД сохранятся в томе)
docker compose down

# Полная очистка (удалит БД и образы)
docker compose down --rmi all -v

# Проверить размер тома БД
docker system df -v | grep postgres_data
```
